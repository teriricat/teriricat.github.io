Hexo博客更新常用操作

Created2021-12-03|Updated2022-10-10|Teririの日常
|Post View:

软件与用途

typora

hexo的博客文件使用的是markdown语法的.md文件。

typora则是最好用的Markdown文件编辑器没有之一！

敲markdown文档的时候可以边敲边预览样式，也可以直接快捷键切回源代码模式。还有一些文件格式的快捷键，尤其是表格插入及其的好用！

sublime

各种乱七八糟格式的文件的打开方式，在hexo中主要用于修改_config.yml与_config.melody.yml两个配置文件

如果你遇到什么文件后缀名很奇怪没有可以查阅编辑的软件，那么sublime一定可以完成你的目的！

Git Bash

github的账号与ssh的绑定，一般配置过一次就用不到下一次了。

划重点

  • 要注意英文的冒号、横杠等符号不要在中文输入法下输入，原因大概是机器认识的符号都是英文的（平时我编写代码也是如此），中文符号在机器的字典里它翻译不了，会报错。

  • 善用快捷键，在Typora编辑器中常用的快捷键如下（从上到下我自己的使用频率从高到低）：

按键 功能
选中文字CTRL+B 加粗
选中文字CTRL+SHIFT+`(数字1左边那个) 注释名词
CTRL+（?/） 来回切换源代码与呈现效果的样式
CTRL+F 查询字符（大多数浏览器网页、编辑器中都可以用此快捷键）

  • cd命令用于终端进入/返回目录层级，终端中的命令直接按上下箭头可以返回到之前输入过的命令

markdown常用语法

  • markdown是一种文本编辑语法，.md后缀名是该文件的存储格式，类似于xlsx，docx，建议将typora设置为.md文件的默认打开方式

    更改typora为.md的默认打开方式：选中某个.md文件右键属性 -> 点击打开方式->更改选择为typora，应用即可。

  • 标题的级数：建议用三级（###）和四级（####）

    需要注意的是 “>”, “-“, “1.”以及标题符号”#”后面都要加一个空格，再拟写文字。

这是一个注解

  • 不排序分点
  1. 排序分点
  • 字体加粗
  • 斜体
  • 下划线
  • 删除横线
  • 注释名词
item1 item2
data1 data2

更新过程

hexo n/ new 字符

作用：在X:\blogdir\source\_posts创建一篇名称为字符.md的文件，会自动生成blog的相关信息

字符.md的文件中会自动生成blog的相关信息，可自定义tags/categories，此处需要注意的是，tags是系统自动生成的，categories字段需要自己添加，英文冒号后面一定要有个空格再填写标签/分类字段

hexo g / hexo generate

作用：将X:\blogdir\source\_posts中的文件渲染成网页形式输出，生成X:\blogdir\public文件夹

与此有一个对应的命令hexo clean将生成的public文件夹删掉（和手动邮件删除该文件夹无差，会用到这个命令是因为有的时候hexo g会导致上一次的报错缓存未清楚掉），就需要先用hexo clean再hexo g

使用场景：X:\blogdir\source\_posts中的博客源文件有更新或者配置文件中_config.yml的样式配置有更新

hexo d / hexo deploy

作用：将public中更新的网页内容上传到自己的github仓库中，以实现博客链接中的内容更新

使用场景：hexo g生成了新的X:\blogdir\public文件夹内容，需要上传本次更新到博客网站中

hexo s / hexo server

作用：将public中更新的网页内容在本地预览（通常是http://localhost:4000 用浏览器打开即可，按Ctrl+C中止本地预览）

Ctrl+C可以中止在终端的各种命令进行的过程（不要滥用即可，有的过程中断了会出现一些不好解决的问题）

使用场景：hexo g生成了新的X:\blogdir\public文件夹内容且有较大更新，需要再上传前确认
